Installation von Programmen bzw. Befehlen

Direkt nach der Installation bringen aktuelle Linux-Distributionen bereits viele Programme mit. Sollte etwas fehlen, so stehen verschiedene Möglichkeiten der Nachinstallation zur Verfügung.

Anwendungsmanager

Komfortable GUI-basierte Anwendungen zum Finden und Nachinstallieren von Programmen stehen eigentlich in jeder Distribution bereit. Man sucht über Kategorien oder mittels Suchfunktion, wählt das gewünschte Programm aus und startet die Installation. Also auch nicht anders als bei anderen Betriebssystemen.

Installation mit apt-get

Was aber, wenn man das Gesuchte dort nicht findet, oder einfach nur schnell einen Befehl für die Bash benötigt, der noch nicht installiert ist?

Die Bash öffnen und - exemplarisch am Befehl whois erklärt - folgenden Befehl eingeben:

sudo apt-get install whois

Auf Deutsch übersetzt: Als Administrator (sudo = „superuser do“) das Programm apt-get nutzen, etwas installieren (daher install) und zwar den Befehl bzw. das Programm whois. Genauso kann man auch egrep, ngrep, etc. installieren.

Installation von Debian-Paketen

Lädt man die Software direkt von der Herstellerwebseite herunter, kann es gut sein, dass man ein Debian-Paket (.deb) erhält. In manchen Fällen reicht das Anklicken der heruntergeladenen Datei, und schon öffnet sich der Anwendungsmanager und man bekommt eine geführte Installation über eine grafische Benutzeroberfläche.

Ansonsten öffnet man die Bash, navigiert in das Verzeichnis in dem das Debian-Paket liegt und gibt

sudo dpkg -i Dateiname.deb

ein.

Das -i steht für „installieren“, danach kommt die zu installierende Datei. Sollte man gleich mehrere Debian-Pakete „in einem Rutsch“ installieren müssen:

dpkg -i *.deb

Durch den Platzhalter " * " werden alle Dateien ausgewählt, die die Suffix „.deb“ haben.

Probleme mit Abhängigkeiten

Ich selbst habe es äußerst selten erlebt, dass es Probleme mit Abhängigkeiten gibt, weil - meines Erachtens nach - Linux da sehr gut und sauber arbeitet. Erforderliche Abhängigkeiten wurden erkannt und die nötigen Sachen gleich mitinstalliert.

Als ich unter elementaryOS 0.4 allerdings TeamViewer installieren wollte, wurde ich mit diesem Problem konfontiert. Ein Öffnen des Debian-Pakets mit dem Anwendungsmanager erbrachte keinen Erfolg, und so installierte ich TeamViewer über die Kommandozeile. Die Abhängigkeiten wurde dummerweise nicht erfüllt, was ich in Form einer Fehlermeldung angezeigt bekam. Wie nun das Problem lösen?

Nach einem kurzen Blick in die manpage von apt-get war die Lösung schnell gefunden:

sudo apt-get install -f

Durch den Parameter "-f" versucht apt-get fehlerhafte Abhängigkeiten zu korrigieren (f = fix broken). Und siehe da: es hat geklappt!
