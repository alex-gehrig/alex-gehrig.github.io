git-Basics

Versionsverwaltung und -kontrolle ist immer ein Thema, wenn man an größeren Projekten arbeitet. Wobei es natürlich höchst individuell ist, was "größer" bedeutet. Verschiedene Systeme stehen zur Auswahl, wobei das am weitesten verbreitet "git" sein dürfte.

Wer sich intensiver mit diesem Thema befassen möchte, dem sei das "Pro Git book" empfohlen: https://git-scm.com/book/en/v2

Im folgenden möchte ich versuchen, einen kleinen Überblick über Standard-Situationen und die dazugehörigen Befehle für die Verwendung auf der Linux-Bash zu geben. Zunächst brauchen wir ein Verzeichnis, in dem wir die Daten unseres Projekts speichern wollen. Dann muss man festlegen, dass eine Versionskontrolle mit git erfolgen soll. Im Anschluss daran brauchen wir dann Befehle für das Hinzufügen einzelner Dateien zur Versionskontrolle, das Anfertigen der Commits, also der "Schnappschüsse" wenn man das so bezeichnen möchte, etc.

Grundsätzliche Einstellungen

  • Eigenen Namen festlegen:

    git config --global user.name "John Doe"

  • Eigene Email-Adresse festlegen:

    git config --global user.email johndoe@example.com

Ohne "--global" werden die Festlegungen nur für den aktuellen Ordner, also das aktuelle Projekt vorgenommen.

Arbeitsschritte im Projekt

  • Versionskontrolle aktivieren:

    git init

  • Den aktuellen Stand abrufen:

    git status

  • Eine Datei namens datei.py für den nächsten Commit vormerken:

    git add datei.py

    ...oder alternativ mehrere Dateien per Wildcard:

    git add *.py

  • Einen Commit mit dem Hinweistext "Mein Commit" vornehmen:

    git commit -m "Mein Commit"

  • Den Commit mit einem online verfügbaren Projekt (Branch = "master") synchronisieren:

    git push https://www.meindienst/mein-benutzername/mein-projekt master

  • ...oder noch einfacher:

    git push origin master

  • Von einem anderen PC die nun neuen Daten aktualisieren:

    git pull

Nach dem Erstellen eines Repositorys

  • Repository (=online) mit lokalem Verzeichnis "verknüpfen":

    git remote add origin https://meindienst/mein-benutzername/mein-projekt.git

Neuer Name für den Standard-Branch

Der Standard-Branch heißt nun 'main' und nicht mehr 'master'. Dies führt beim pushen zu Problemen. Daher sind folgende Schritte einmalig erforderlich - das lokale Repository hat nämlich immer noch 'master' als Standard-Branch.

  • lokalen Branch in 'main' umbenennen:

    git branch -M main

  • den Branch 'main' für den Upstream festlegen:

    git push -u origin main

Ab jetzt kann man wie bisher weiterarbeiten.
